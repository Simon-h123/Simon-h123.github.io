安装hexo的记录过程

ubuntu18.0安装hexo的记录过程

这篇博客主要用来记录在ubuntu18.0下安装hexo需要提前准备的环境、安装过程中出现的问题以及它的处理方法、hexo本地地址预览、如何使用github部署hexo到远端、github部署过程中出现的问题以及解决方法，最后记录了一些hexo基本使用命令

安装hexo需要提前准备的环境

nodejs 中文官网下载地址

我安装的方法是使用直接使用官方安装包的方式，安装的版本是node-v14.21.3-linux-x64(包含了需要用到的node和npm)，具体安装步骤如下

1. 在官网下载安装包， 注意是.xz 格式

2. 解压压缩包并创建node和npm的软链接

解压压缩包命令：

    xz -d node-v14.21.3-linux-x64.tar.xz
    tar xvf node-v14.21.3-linux-x64.tar

创建软链接：

    sudo ln -s 安装包位置/bin/node /usr/local/bin/
    sudo ln -s 安装包位置/bin/npm /usr/local/bin/

查看是否安装上：

    node -v<br />
    npm-v<br />

官网更多版本下载的网址

3. 更换镜像源

利用npm下载cnpm ：

    npm install -g cnpm --registry=http://registry.npm.taobao.org<br />

查看安装版本：

    cnpm-v

4. 安装hexo框架

利用cnpm来安装：

    cnpm install -g hexo-cli

查看安装版本： 

hexo -v

安装过程中遇到的问题

安装过程中会遇到各种版本冲突问题，这个需要根据自己系统的来看，建议使用npm6.9及以上的版本,实在不行的话就卸载了再重装

附卸载nodejs的命令：

sudo apt-get remove --purge npm
sudo apt-get remove --purge nodejs
sudo apt-get remove --purge nodejs-legacy
sudo apt-get autoremove

同时，还需要删除对应的软链接

sudo rm -rf /usr/local/bin/node
sudo rm -rf /usr/local/bin/npm

查看node与之相匹配的npm的网址

hexo本地地址预览

先创建一个用于博客的空文件夹，例如 blog；然后，进入blog文件夹；初始化生成博客；最后，命令行启动hexo本地预览

mkdir blog
cd ~/blog
hexo init
hexo s

此时，会在终端返回一个网址，在终端打开就可以看见一个hello world的初始博客，blog文件会包含博客使用的所有文件，博客文章所在的目录为blog/source/_posts/中，一个.md文件为一个博文文件。

本地4000打不开时，可修改_config.yml文件，替换4000端口

打开_config.yml，在_config.yml文件末尾添加以下内容

gedit _config.yml
    server:
        port: 4001 #端口号随意，默认是4000
        compress: true
        header: true

在浏览器中输入：

    localhost:4001

如下显示，表示hexo框架已经搭建好一半了

部署远程到github

远程部署hexo到github大概分为几个步骤，首先需要在我们的github上创建一个空的仓库，注意：仓库名一定要是 你的用户名.github.io

然后下载一个git部署插件

    cnpm install --save hexo-deployer-git

后打开_config.yml文件，在文件末尾添加以下：

    # Deployment
    ## Docs: https://hexo.io/docs/one-command-deployment
    deploy:
        type: 'git'
        repo: https://github.com/你的github用户名/你的仓库名.git
        brach: master

也可在下图位置复制url到repo处

然后进行远程部署：

    hexo d

终端执行hexo d需要进行身份验证，会提示输入你的github用户名以及密码验证，但是2021.3月以后github在push时不能再用登陆密码进行验证，因此我们需要为你的仓库设置一个ssh密钥，或者token，这样，输入密码验证时输入对应的ssh密钥即可(以下介绍密钥的生成与使用)

配置github的Deploy keys

首先在终端配置ssh

    git config --global user.name"你的github用户名"
    git config --global user.email"你的github登陆邮箱"
    ssh-key-gen -t rsa -C"你的github登陆邮箱"

终端查看配置的ssh

     cat ~/.ssh/id_rsa.pub

然后，在刚刚创建的github仓库中生成github的deploy keys


浏览器输入 “创建仓库名”即可进入你的博客
